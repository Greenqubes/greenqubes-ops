# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "Claude Warm" dark mode toggle to the UserMenu that persists in localStorage and auto-detects the device system preference on first visit.

**Architecture:** `next-themes` manages theme state, localStorage persistence, system preference detection, and flash prevention. A `.dark` class is added to `<html>` by the library; dark token values are defined in `globals.css` under `.dark {}`. The toggle button lives in UserMenu, which reads and sets theme via `useTheme()`.

**Tech Stack:** next-themes, Tailwind CSS (CSS variables), Next.js App Router, lucide-react

## Task 2: Wire ThemeProvider into the app layout

**Files:**
- Create: `src/components/ThemeProvider.tsx`
- Modify: `src/app/layout.tsx`

Next.js App Router server components can't use React context directly. The pattern is to create a thin `'use client'` wrapper that re-exports `NextThemesProvider` with the app's config baked in, then import that wrapper into the server-component layout.

- [ ] **Step 1: Create the ThemeProvider wrapper**

Create `src/components/ThemeProvider.tsx`:

```tsx
'use client'

import { ThemeProvider as NextThemesProvider } from 'next-themes'

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  return (
    <NextThemesProvider attribute="class" defaultTheme="system" enableSystem>
      {children}
    </NextThemesProvider>
  )
}
```

- `attribute="class"` — next-themes adds/removes `.dark` on `<html>`
- `defaultTheme="system"` — first visit uses the device system preference
- `enableSystem` — activates system preference detection

- [ ] **Step 2: Update layout.tsx**

Replace the contents of `src/app/layout.tsx` with:

```tsx
import type { Metadata } from 'next'
import { Fraunces, IBM_Plex_Sans } from 'next/font/google'
import { Suspense } from 'react'
import { ThemeProvider }          from '@/components/ThemeProvider'
import { ToastProvider }          from '@/components/Toast'
import { ErrorBoundary }          from '@/components/ErrorBoundary'
import { FloatingChatWrapper }    from '@/components/FloatingChatWrapper'
import { BugReportButton }        from '@/components/BugReportButton'
import './globals.css'

const fraunces = Fraunces({
  subsets: ['latin'],
  variable: '--font-fraunces',
  display: 'swap',
  axes: ['opsz'],
})

const ibmPlexSans = IBM_Plex_Sans({
  subsets: ['latin'],
  weight: ['400', '500', '600'],
  variable: '--font-ibm-plex-sans',
  display: 'swap',
})

export const metadata: Metadata = {
  title: 'Greenqubes Ops',
  description: 'Internal operations platform',
}

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={`${fraunces.variable} ${ibmPlexSans.variable}`} suppressHydrationWarning>
      <head>
        {/* CJK + Bengali: system fonts handle these on all target devices */}
      </head>
      <body className="font-body antialiased">
        <ThemeProvider>
          <ErrorBoundary>
            <ToastProvider>
              {children}
              <BugReportButton />
            </ToastProvider>
            <Suspense fallback={null}>
              <FloatingChatWrapper />
            </Suspense>
          </ErrorBoundary>
        </ThemeProvider>
      </body>
    </html>
  )
}
```

Key changes from the original:
- `ThemeProvider` imported and wraps `ErrorBoundary` + everything inside `<body>`
- `suppressHydrationWarning` added to `<html>` — next-themes mutates the `class` attribute before React hydrates, which would otherwise log a harmless warning. This suppresses it.

## Task 3: Add dark mode toggle to UserMenu

**Files:**
- Modify: `src/components/UserMenu.tsx`

- [ ] **Step 1: Add imports**

At the top of `src/components/UserMenu.tsx`, add to the existing lucide-react import line:

```tsx
import { LogOut, ShieldCheck, LayoutDashboard, Languages, Eye, EyeOff, Moon, Sun } from 'lucide-react'
```

And add the next-themes import after the existing imports:

```tsx
import { useTheme } from 'next-themes'
```

- [ ] **Step 2: Add theme state inside the component**

Inside `UserMenu`, directly after the existing `useState`/`useRef` declarations, add:

```tsx
const { resolvedTheme, setTheme } = useTheme()
const [mounted, setMounted] = useState(false)
```

Then add a `useEffect` (after the existing ones) to mark the component as mounted:

```tsx
useEffect(() => { setMounted(true) }, [])
```

The `mounted` guard is required because `resolvedTheme` is `undefined` on the server (next-themes only knows the theme after reading localStorage in the browser). Without this guard, the icon and label would mismatch on first render.

- [ ] **Step 3: Add the toggle button to the dropdown**

In the JSX, find the language switcher section which ends with:

```tsx
          </div>
        </div>

          {/* Admin shortcuts */}
```

Add the dark mode toggle row between them (after the language section's closing `</div>` and before the Admin shortcuts comment):

```tsx
          {/* Dark mode toggle */}
          {mounted && (
            <button
              onClick={() => setTheme(resolvedTheme === 'dark' ? 'light' : 'dark')}
              className="w-full flex items-center gap-2.5 px-4 py-2.5 text-sm text-ink2 hover:bg-bg hover:text-ink transition-colors border-t border-line"
            >
              {resolvedTheme === 'dark' ? (
                <Sun size={14} strokeWidth={1.8} />
              ) : (
                <Moon size={14} strokeWidth={1.8} />
              )}
              {resolvedTheme === 'dark' ? 'Light mode' : 'Dark mode'}
            </button>
          )}
```

The `{mounted && ...}` wrapper means the button only appears once the browser has read localStorage — this prevents a flicker where the wrong icon shows for a frame.
