# Assistant History Sidebar — Design Spec

_Written: 2026-05-11_

---

## Overview

Add a persistent conversation history sidebar to the `/assistant` page. Users can browse their last 30 chats, resume any conversation, pin up to 5 for quick access, and delete conversations permanently.

---

## Layout

### Desktop

- Persistent left sidebar (~260px wide) beside the chat area
- Always visible — no toggle
- Sidebar sections (top to bottom):
  1. **Pinned** section — shown only when at least one chat is pinned
  2. **Today** group
  3. **This Week** group
  4. **Earlier** group
- The list is scrollable; all 30 chats are reachable by scrolling
- "New Chat" button pinned to the bottom of the sidebar

### Mobile

- Sidebar is hidden entirely on small screens
- A **History** icon button appears in the `/assistant` page header
- Tapping it navigates to `/assistant/history` — a full-screen list with the same grouping
- Tapping a chat on `/assistant/history` navigates to `/assistant?chat=<id>`, loading that conversation
- Back button on `/assistant/history` returns to the current chat

---

## Each History Row

Each row shows:
- **Topic** (from `asst_chats.topic`, auto-generated by Haiku tagger on save)
- **Message count** (derived from `msgs` array length)
- **Importance stars** (from `asst_chats.importance`, 1–5)
- A pin indicator (📌) when pinned

---

## Interactions

### Loading a past conversation

Clicking/tapping a history row loads the `msgs` array from that `asst_chats` row into the chat area. The user can read and continue from where the conversation left off. Sending a new message continues with the full history as context. On navigate-away or "New Chat", the resumed + extended conversation saves as a **new** `asst_chats` row. The original row is not modified.

### Desktop — hover interactions

Hovering a row reveals two icons on the right:
- **Pin icon** — click to pin immediately (or unpin if already pinned). No dropdown needed.
- **⋮ icon** — click to open a small dropdown with "Delete Conversation"

### Mobile — ⋮ dropdown

Each row has a visible ⋮ icon on the right. Tapping it opens a dropdown with:
- **Pin** (or **Unpin** if already pinned)
- **Delete Conversation**

### Pinning

- Max 5 pinned conversations
- Pinned chats appear in a "Pinned" section at the top of the sidebar (and top of the mobile list)
- If already at 5 pins and user tries to pin a 6th: show a toast — "You can pin up to 5 conversations"
- Pinned state stored in DB (new `pinned` boolean column on `asst_chats`, migration 0015)
- Unpin by clicking the pin icon again (desktop) or via ⋮ → Unpin (mobile)

### Deleting

- Tapping "Delete Conversation" shows a confirmation prompt: **"Delete Permanently?"** with Confirm and Cancel buttons
- On confirm: hard delete from `asst_chats` (full row removal, no soft delete)
- Immediately removed from the sidebar list
- If the deleted conversation was the currently active chat, the chat area resets to the empty new-chat state

## Data

### DB change

New migration (0015): add `pinned boolean NOT NULL DEFAULT false` to `asst_chats`.

### Existing queries used

- `getRecentChats(30)` in `src/lib/supabase/queries/assistant.ts` — already exists. Needs query updated to `ORDER BY pinned DESC, ts DESC` so pinned rows always surface at the top even when fetching the 30 most recent.

### New API routes

| Route | Method | Purpose |
|---|---|---|
| `/api/assistant/history` | `GET` | Fetch last 30 chats for the current user |
| `/api/assistant/pin` | `PATCH` | Toggle pinned state on a chat (`{ id, pinned }`) |
| `/api/assistant/delete` | `DELETE` | Hard delete a chat by id |

## Constraints

- Pin cap: 5. Enforced in both UI (toast) and API (check before insert).
- Hard delete only — no soft delete, no recovery.
- Date labels (Today / This Week / Earlier) always in English regardless of user language setting.
- No search bar in this version.
- Floating chat panel history: out of scope — future session.
