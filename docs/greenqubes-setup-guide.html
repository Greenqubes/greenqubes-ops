Workstation
Windows PC · where you develop
Core tools — install in this order
1
Git Free One-time
The tool that talks to GitHub — saves versions of your code and pushes changes to the cloud. Install with all default settings, nothing to configure.
git-scm.com/download/win # After install, open Command Prompt and verify: git --version
2
Node.js (LTS version) Free One-time
The engine that runs JavaScript on your computer. Required for Claude Code and for running the app locally during development. Pick the LTS version — not "Current."
nodejs.org # Verify install: node --version npm --version
3
VS Code Free One-time
Your main working environment. A code editor with a proper file browser, terminal, and a Claude Code sidebar — much friendlier than a bare terminal. Download and run the installer, all defaults are fine.
code.visualstudio.com
4
Claude Code Free to install
Install two ways — as a VS Code extension (for the sidebar chat panel) AND as a global CLI tool (so it runs in the terminal). Do both.
# In VS Code: Extensions panel (left sidebar) # Search "Claude Code" → Install # In terminal (installs the CLI globally): npm install -g @anthropic-ai/claude-code # Verify: claude --version
Version control
5
GitHub Desktop Free Optional but recommended
A visual app for pushing code to GitHub. Makes the "push to GitHub" step a big obvious button instead of a terminal command. Good while you're getting comfortable with the workflow. Sign in with your GitHub account.
desktop.github.com
Account setup — do these online
6
GitHub account + repo Free
Sign up at github.com with your work email. Create a new private repository called greenqubes-ops. Enable 2FA (two-factor authentication) immediately — Settings → Password and authentication.
github.com # After creating the repo, clone it locally: cd C:\Users\YourName\projects git clone https://github.com/yourname/greenqubes-ops.git cd greenqubes-ops
7
Vercel account Free to start $20/mo when live
Sign up at vercel.com using your GitHub account — one-click OAuth, no separate password. Connect it to your greenqubes-ops repo. Every push to main branch auto-deploys. Enable 2FA.
vercel.com
8
Supabase account + project Free to start
Sign up at supabase.com. Create a new project — region: Southeast Asia (Singapore) ap-southeast-1. Save the database password in a password manager. Copy your Project URL and anon key — you'll paste these into Vercel's environment variables later.
supabase.com
9
Anthropic API account Pay-per-use
Sign up at console.anthropic.com — separate from your Claude.ai account even though it's the same company. Add a credit card and $10 initial credit for testing. Generate an API key and save it. Enable 2FA.
console.anthropic.com
10
Voyage AI account ~$1–5/mo
Sign up at voyageai.com. Add a credit card (negligible cost at our scale — cents per month during development). Generate an API key and save it. Anthropic's recommended embedding partner.
voyageai.com
11
Cloudflare account R2 free up to 10GB Images $5/mo
Sign up at cloudflare.com. Create an R2 bucket named greenqubes-files. Separately activate Cloudflare Images. Generate an R2 API token with read/write permissions. Save your Account ID and token. Enable 2FA.
cloudflare.com
12
Telegram Bot Free
Open Telegram on your phone. Search @BotFather. Send /newbot and follow the prompts. Name it something like "Greenqubes Ops." Save the bot token BotFather gives you.
Local tools — knowledge base
13
Obsidian Free One-time
Download and install at obsidian.md. Create a new vault called greenqubes-kb — put it in a folder you'll remember, e.g. C:\projects\greenqubes-kb. This is where your team writes SOPs, client notes, and supplier info.
obsidian.md
14
Obsidian Git plugin Free Optional but recommended
Inside Obsidian: Settings → Community Plugins → Browse → search "Obsidian Git" → Install → Enable. This auto-syncs your vault to a private GitHub repo every time you save a note. Keeps everything backed up and makes it available to the server for nightly indexing.
Save all your keys
15
Create .env.local file One-time
Once Claude Code has scaffolded the project, I'll create this file for you. You paste your saved keys into it. It never gets pushed to GitHub — git ignores it automatically. Then push a copy to Vercel's environment variables panel.
SUPABASE_URL=https://xxx.supabase.co SUPABASE_ANON_KEY=eyJ... SUPABASE_SERVICE_KEY=eyJ... ANTHROPIC_API_KEY=sk-ant-... VOYAGE_API_KEY=pa-... CLOUDFLARE_ACCOUNT_ID=abc123 R2_ACCESS_KEY_ID=... R2_SECRET_ACCESS_KEY=... CLOUDFLARE_IMAGES_TOKEN=... TELEGRAM_BOT_TOKEN=123456:ABC...

Server PC
Office Windows PC · backup only · never for development
Hardware first
H1
External SSD or HDD ~S$80–150 once
Minimum 2TB. Recommended: Samsung T7 (SSD, fast, compact) or WD My Passport (HDD, cheaper, larger). Plug into the server PC via USB. Format as NTFS. Label it physically: "Greenqubes Backup — Do Not Format." Assign it drive letter D:\ in Windows Disk Management.
Software — install in this order
S1
rclone Free One-time
The one program that does all the backup work. Downloads a single .exe file — no installer wizard. Drop it in C:\rclone\ and add that folder to Windows PATH so you can run it from anywhere.
rclone.org/downloads # Verify in Command Prompt: rclone --version
S2
Configure rclone → Cloudflare R2 One-time
Run the setup wizard and connect rclone to your Cloudflare R2 bucket. I'll walk you through every prompt when we get here — you'll need your R2 Account ID, Access Key and Secret Key from Cloudflare's dashboard. Takes about 10 minutes.
# Starts the config wizard: rclone config # When done, test the connection: rclone ls r2:greenqubes-files
S3
Backup batch script (.bat) Free I write this for you
A simple script file I create during the Claude Code setup session. Save it at C:\Scripts\greenqubes-backup.bat. You never need to open or edit it. It does three things every night: copies R2 files to external drive, indexes new Obsidian notes to Supabase, writes a log.
# What the script does internally (I write this): rclone copy r2:greenqubes-files "D:\Greenqubes-Archive" ^ --immutable ^ --log-file="D:\backup-log.txt" node C:\Scripts\obsidian-sync.js # (sends new Obsidian notes to Supabase for AI search)
S4
Windows Task Scheduler Already on Windows One-time setup
Already installed on every Windows PC — no download needed. You open it, point it at the backup script, set it to run at 2am every day. After that, never touch it again. The PC must be left on overnight for this to work.
# Open Task Scheduler: taskschd.msc # Create Basic Task → name it "Greenqubes Nightly Backup" # Trigger: Daily at 2:00 AM # Action: Start a program # Program: C:\Scripts\greenqubes-backup.bat # Finish. Done forever.
S5
Node.js (LTS) Free One-time
Same as workstation — needed to run the Obsidian-to-Supabase sync script. Same download, same installer, same defaults.
nodejs.org # Verify: node --version
Obsidian vault on the server
S6
Git + clone the vault repo Free One-time
Install Git on the server PC (same download as workstation). Then clone the private greenqubes-kb repo. The nightly script does a git pull before indexing, so it always processes the latest notes you've written on your MacBook or workstation.
# One-time setup: git clone https://github.com/yourname/greenqubes-kb.git C:\greenqubes-kb # Nightly (inside backup script, I write this): cd C:\greenqubes-kb && git pull
Keep the PC healthy
S7
Windows power settings Already on Windows One-time
Set the PC to never sleep and never turn off the hard drive automatically. Otherwise Task Scheduler can't run the 2am backup. Control Panel → Power Options → Change plan settings → set all to "Never."
# Or via PowerShell (run as admin): powercfg -change -standby-timeout-ac 0 powercfg -change -disk-timeout-ac 0
S8
Windows Remote Desktop Already on Windows Optional
Enable Remote Desktop so you can check or control the server PC from your workstation or MacBook without physically walking over to it. Settings → System → Remote Desktop → Enable. Set a static local IP on the server so the address doesn't change.
# Enable via PowerShell (run as admin): Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -name "fDenyTSConnections" -Value 0 Enable-NetFirewallRule -DisplayGroup "Remote Desktop"
S9
That's it for the server Nothing else needed
The server PC has no other job. No development tools, no code editor, no GitHub Desktop, no Obsidian app (the vault is just a folder of text files — Obsidian app is only needed on machines where people write notes). After setup, you check on it via the infrastructure dashboard in the web app — not by physically touching the machine.
